HTTP/2 → HTTP/3: Modern Performance

From SPDY to HTTP/2 (2015)

Google’s SPDY (2009) inspired HTTP/2, which introduced binary framing, multiplexing, header compression, prioritization, and server push. Binary framing breaks messages into frames so multiple resources can be fetched concurrently on a single TCP connection.

HTTP/3 + QUIC (2020)

  • QUIC over UDP: Faster handshakes, integrated TLS 1.3, and better mobility across networks.
  • No Transport HOL Blocking: Loss on one stream doesn’t stall others thanks to stream isolation.
  • Real-world wins: More resilient on lossy/mobile networks; widely supported by major browsers and platforms.

What is QUIC (Simple temrs)?

Think of TCP like sending certified mail — reliable but slow because each step needs a signature. QUIC is more like secure messaging — it’s encrypted by default and recovers from hiccups quickly, so the conversation keeps flowing.

Adoption continues to grow. Some legacy firewalls/servers lag behind and UDP debugging can be harder, but the trajectory is toward HTTP/3 by default.
