HTTP/0.9 → HTTP/1.1: Origins & Growth

Sir Tim Berners-Lee created HTTP, HTML, the first web server, and the first web browser. HTTP/0.9 started as a minimal, text-only protocol. By 1996, HTTP/1.0 added headers and status codes to improve communication, though performance suffered due to one request per connection as pages became more complex.

HTTP/1.1 (1997)

  • Persistent connections enable multiple requests over the same TCP connection.
  • Pipelining allows sending multiple requests before responses (limited adoption in practice).
  • Improved caching rules and required Host headers (virtual hosting on shared IPs).

HTTP/1.1 became widely deployed and is still supported today, laying the foundation for future versions and the growth of the web.
