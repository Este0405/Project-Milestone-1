Key Concepts

Request / Response
The browser (client) sends a request; the server responds with resources like HTML, images, and video.
Statelessness
Each HTTP request is independent; cookies/sessions provide continuity for auth and personalization.
Persistent Connections
HTTP/1.1 keeps TCP connections open for multiple requests, reducing repeated handshakes.
Multiplexing
HTTP/2 interleaves multiple streams over one connection to improve page load times.
Head-of-Line (HOL) Blocking
Packet loss can stall delivery of subsequent data. QUIC mitigates this by isolating streams.
QUIC
A secure, UDP-based transport that integrates TLS 1.3, fast handshakes, loss recovery, and connection migration.
Connection Migration
QUIC’s connection IDs allow a session to survive network changes — helpful on mobile.
